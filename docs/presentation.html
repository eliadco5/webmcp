The Problem

How agents work today — and what it costs

Every subsequent tool call re-reads all prior results. Cost compounds.

3-call pattern (status quo)
1. searchAvailability
↓ agent reads result #1
2. createReservation
↓ agent re-reads results #1 + #2
3. addLoyaltyPoints
3 HTTP calls · growing context
AgentBridge composite (this protocol)
1. book({ guestId, roomId, dates })
↓ orchestration runs server-side
← structured result, zero cumulative context
1 HTTP call · no context bleed
The agent calls one named function. Domain logic lives in the page — not as tokens in the agent's context.

  • Cumulative context is the hidden cost: each new tool call re-reads all prior results. A 3-step workflow with average output tokens of 614 each ≈ 3,251 cumulative tokens — the single-call approach contributes 0.
  • Domain logic as tokens means the agent must know sequencing, preconditions, and error handling — fragile and expensive. The composite op moves that logic into the page.

Progressive Tool Disclosure

Agents discover operations without loading all 55

Path A — Load & call natively
1. explore("reservation.*") → subtree
2. load_tools(["book"]) → session state
3. Call book(…) as a native tool

Tool state persists per-token, 24h TTL. Agent sees only what it loaded.

Path B — Invoke directly (no state)
1. invoke("book", {…})
← result returned inline

One-off calls without session overhead. Reads dispatched concurrently; writes sequentially.

Security & Access Control

Layered controls at every call boundary

Role is checked at register time and on every call. Destructive ops carry requiresConfirmation: true — UI shows a dialog; agent must pass confirm: true.

  • RFC 8707 audience-bound tokens prevent token reuse across services — a token minted for this MCP server cannot be replayed elsewhere.
  • Capability hash lets agents detect registry changes between sessions and refresh their tool list automatically.
Next Steps

What we're proposing

  • Adopt the Operation registry pattern as the standard integration layer — single source of truth for both browser and server agent surfaces.
  • Prefer composite ops over raw tool chains — move domain orchestration into the page, not into agent prompts.
  • Use progressive disclosure (explore → load_tools or invoke) to keep context budgets flat regardless of registry size.
  • Enable server-side in-process dispatch (makeDispatch(ctx)) for the maximum latency win — 3× faster than HTTP sub-calls.
